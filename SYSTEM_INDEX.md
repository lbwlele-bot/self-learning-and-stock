# 系统总索引

这套系统的定位是：
`主题优先导航 + 三层骨架约束的投资决策辅助系统`

它不是自动荐股系统，也不是纯学习笔记库。
它的目标是把：
`逻辑基点 -> 工作假设 -> 投资翻译 / 执行`
这条链条跑顺，并且让高频入口足够清晰。

## 最小阅读路径

如果是新对话或重新接入系统，只读这 5 个：

1. `START_HERE.md`
2. `SYSTEM_INDEX.md`
3. `TOPIC_MAP.md`
4. `memory/logic-bases.md`
5. `memory/context.md`

如果要补齐规则层，再加这 2 个：

6. `memory/principles.md`
7. `memory/open-questions.md`

## 怎么理解这套结构

- `主题仍是主导航`
  日常继续先从 `TOPIC_MAP.md` 找当前在讨论什么。
- `三层结构是底座`
  每个主题都应能追溯到：
  `逻辑基点 -> 工作假设 -> 投资翻译 / 执行`
- `playbooks 是执行层`
  只在逻辑和假设相对清楚后使用。

一句话：
`先用主题找到问题，再用三层结构约束讨论深度和落地路径。`

## 唯一主文件

下列概念以后只认一个主文件：

- 系统总导航：`SYSTEM_INDEX.md`
- 新对话启动：`START_HERE.md`
- 最小入口清单：`ESSENTIALS.md`
- 当前系统状态：`CURRENT_SYSTEM.md`
- 文件地图：`SYSTEM_MAP.md`
- 逻辑基点：`memory/logic-bases.md`
- 阶段摘要：`memory/context.md`
- 原则：`memory/principles.md`
- 开放问题：`memory/open-questions.md`
- 主题进度：`TOPIC_MAP.md`
- 发现路径：`docs/frameworks/DISCOVERY_PIPELINE.md`

规则：

- 新内容先判断归属，再更新主文件
- 非主文件只做引用、跳转、极短摘要
- 不再在多个总览文件里完整重讲同一个定义
