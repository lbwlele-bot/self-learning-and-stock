# 瑞芯微长期观察卡

## 1. 为什么进入观察名单

- 不是因为“AI概念很热”
- 而是因为在真实客户支持里，已经感受到 RK3588 在端侧 AI / AIoT 项目中的客户粘性和替代阻力
- 这说明它的竞争力至少已经部分体现在真实方案落地中，而不只是停留在宣传层

## 2. 我押的核心逻辑

- 我押的不是“股价会涨”
- 而是：
  - 瑞芯微在端侧 AI / AIoT 场景里，已经形成了产品竞争力、方案嵌入和迁移成本
  - 这种竞争力有机会继续转化成收入、利润和新平台延续

## 3. 前置信号

### 让我觉得它在变强的变化

- 客户对 RK3588 这类平台的粘性继续增强
- 新项目不只是试用，而是持续上量
- 不只是单点爆款，而是形成产品序列延续
- 端侧 AI 相关场景持续扩张，而不是停在概念阶段

### 让我觉得它在变弱的变化

- 客户开始明显松动，对替代平台接受度提升
- 粘性主要来自短期缺货、价格或偶然因素，而不是产品力
- 新平台承接不上，增长过度依赖单一老产品
- 行业景气在，但公司没有持续兑现

## 4. 后验验证

- 后面重点看：
  - 收入增长是否持续
  - 利润增长是否持续
  - 产品结构是否继续往 AIoT / 高价值场景走
  - 新平台（如 RK3576 等）能否接力
  - 公司是否继续强调汽车电子、机器视觉、工业等高价值场景

## 5. 市场预期

- 当前更像：
  - 已经开始被市场关注
  - 但是否已经把长期竞争力完全打满，还需要继续观察

## 6. 当前定位

- 长期观察
- 候选机会

## 7. 最重要的一句话

- 瑞芯微当前最值得继续盯的一件事：
  - RK3588 带来的客户粘性，能否继续被新平台、新场景和持续商业结果验证，而不是停留在单点产品成功。

## 8. 更细的消息源与前置信号

### 官方但比财报更早

- 公司官网新闻与生态大会材料
- 投资者互动 / 公告中的产品节奏表述
- 股权激励或经营目标中的增长要求

### 产业前置信号

- 开发板厂商、模组厂商是否开始围绕新平台推板卡和方案
- 生态伙伴是否开始做适配、教程、案例、量产方案
- 客户案例是否从“展示”变成“重复出现”
- 招聘方向是否明显往新平台、汽车、端侧 AI、编译器/工具链倾斜

### 开源与开发者信号

- BSP / SDK / RKNN 工具链更新节奏
- GitHub、开源社区、主线内核支持推进
- 开发者论坛里对成熟度、稳定性、兼容性的真实反馈

### 汽车领域额外信号

- 是否出现车规版本或明确车载型号节奏
- 智能座舱 / 车载 AI Box / Tier1 方案里的重复出现
- 汽车相关认证、合作与量产消息

### 需要刻意防噪音

- 单条论坛爆料
- 没有客户名和产品名支撑的“即将量产”
- 只讲峰值算力、不讲功耗、成本、工具链和场景适配的宣传

## 9. 瑞芯微前置信号短清单

以后顺手留意这几条就够，不需要做成重研究：

1. `新平台有没有开始被生态伙伴反复提到`
   - 不是只在瑞芯微自己口中出现
   - 而是板卡、模组、方案商也开始围绕它做事

2. `RK3588 的客户粘性有没有继续被印证`
   - 老客户是否继续复用
   - 新项目是否仍把它作为主平台之一

3. `RK3576 / RK3688 / RK182X 这类新平台是不是从“宣传”走向“可用”`
   - 有没有 demo、方案、适配、量产迹象

4. `开发者生态是否跟上`
   - SDK / BSP / RKNN / 工具链更新是否积极
   - 开发者反馈更偏“能用”还是“坑很多”

5. `汽车与 AIoT 场景有没有重复出现`
   - 不是一次性展示
   - 而是开始反复出现在智能座舱、机器视觉、工业、机器人、边缘 AI 等场景中

6. `公司招聘和组织动作有没有往这些方向倾斜`
   - 新平台、汽车、端侧 AI、工具链、客户支持等岗位是否明显加强

7. `后续财报有没有兑现这些前置信号`
   - 收入和利润能否接住
   - 不是只有故事，最后还得回到商业结果

## 10. 优先观察项

如果只顺手留意最值得的 3 条，优先级先这样放：

1. `客户对 RK3588 的复用和迁移态度`
   - 最能直接体现真实粘性和替代阻力
   - 这也是最接近你一线工作体感的信号

2. `RK3576 / RK3688 / RK182X 是否从宣传走向可用`
   - 看有没有方案商、板卡、模组、客户导入和适配动作
   - 这是判断公司能不能从“单点爆款”走向“平台延续”的关键

3. `开发者生态成熟度`
   - 看工具链、BSP、SDK、RKNN 以及论坛反馈
   - 这会比很多宣传更早暴露平台质量和落地难度

## 11. 当前一线产业体感（待后验验证）

这部分不是财报事实，而是来自真实客户支持场景的产业前置信号。

### 当前护城河的核心表达

- 国内客户有较强的国产替代需求
- RK3588 当前不只是“能替代”，而是同时具备：
  - 更强性能
  - 更低价格
- 这意味着它给客户提供的是明显的 `性价比优势`

### 当前粘性的投资语言

- 客户没有明显迁走理由
- 这种粘性不只是生态粘性，也包含：
  - 国产替代逻辑
  - 性能 / 成本比较
  - 项目切换成本

### 下一阶段最大的观察点

- 风险不在于 RK3588 当前是否能打
- 风险在于：
  - 客户是否有动力从 RK3588 升级到新平台
  - RK3688 性能更强，但如果价格更高、场景不够清晰，客户未必愿意升级

### 这意味着什么

- 当前的强，更多说明 `RK3588 成功`
- 未来是否还能继续强，关键看公司能否把这种成功延续到下一代平台，而不是只靠一代爆款吃红利
