# 潜在承接者发现路径

这份文件是：
`从逻辑基点走到研究对象的唯一主文件`

它负责四件事：

1. 发现路径
2. 类型样本 / 重点研究对象 / 候选交易对象区分
3. 适合研究的公司类型
4. 热度过滤与预期饱和度检查

一句话：
`不是从股票找逻辑，而是从逻辑长出对象，再用市场饱和度把对象筛一遍。`

## 一、发现路径

固定按这 7 步走：

1. `逻辑基点`
2. `工作假设`
3. `受益产业链`
4. `关键环节`
5. `热度过滤`
6. `观察名单`
7. `执行候选`

默认流程：
`逻辑基点 -> 工作假设 -> 受益产业链 -> 关键环节 -> 热度过滤 -> 观察名单 -> 执行候选`

### 1. 从逻辑基点出发

先问：
- 当前最强的逻辑基点是什么
- 这条基点会把注意力指向什么方向

而不是先问：
- 哪只股票最近最热

### 2. 长出工作假设

把世界观压成可验证命题。

例子：
- 高摩擦世界里，安全、替代、韧性需求会上升
- AI 受限条件下，中国更可能在端侧、工程优化、行业应用形成优势
- 市场可能低估某家公司从单点产品走向平台能力的持续性

### 3. 翻译成受益产业链

先落到：
- 行业
- 链条
- 环节

而不是直接落到股票代码。

### 4. 找关键环节

强制问三句：

- 哪个环节少了会疼？
- 哪个环节短期难替代？
- 哪个环节最可能把重要性传导成利润？

### 5. 做热度过滤

这一层必须独立存在。
它的作用是把：
`好公司`
和
`好研究对象`
分开。

### 6. 进入观察名单

通过前面五步后，再进入模板化管理。

默认模板：
`templates/key-link-watchlist-template.md`

### 7. 验证后再进入执行层

最后才讨论：
- ETF 表达
- 个股候选
- 小仓试探
- 加减仓与撤回

## 二、三种对象必须分清

### 1. 类型样本

作用：
- 帮助理解某类公司长什么样

不负责：
- 提供预期差

### 2. 重点研究对象

作用：
- 仍可能存在研究增量
- 值得持续跟踪和验证

### 3. 候选交易对象

作用：
- 不只值得研究
- 当前时点、验证和预期差开始接近可执行

## 三、适合研究的公司类型

### 1. 关键环节里的二线承接者

特征：
- 环节真关键
- 不是市场最熟悉的一线总龙头
- 在细分环节里有卡位价值

### 2. 从单点产品走向平台能力的公司

特征：
- 市场还把它当成一个产品或一代爆款
- 但它已经在长出多产品、多客户、多场景能力

### 3. 替代链里即将越过验证线的公司

特征：
- 已进入客户验证、导入、试单或放量前夜
- 还没完全体现在利润表和市场共识里

### 4. 深度绑定大客户的隐形冠军

特征：
- 不一定在公众叙事中心
- 但深度嵌入核心客户体系
- 替换成本高，业务粘性强

### 5. 稳中带进攻的升级型公司

特征：
- 旧业务已有一定稳定性
- 新业务、新客户或新平台开始长出来

## 四、默认降权的对象

以下对象默认不作为重点预期差研究对象：

- 已被全市场讲透的一线总龙头
- 纯题材、纯概念、验证路径不清的故事股
- 完全靠盘口和情绪接力的机会
- 国家意义很大但难进利润表的公司
- 只是大逻辑边缘映射的概念股

## 五、市场认知与预期饱和度检查

每个候选对象都先过这 5 句：

1. 它是不是已经成了高频热门叙事的一部分？
2. 最近 3 到 6 个月，是不是机构、媒体、散户反复讨论的对象？
3. 市场是不是已经给它贴上了明确标签？
4. 当前股价主要在交易已兑现、正在验证，还是远期想象力？
5. 我现在研究它，是在研究预期差，还是在补大众课？

如果 5 条里有 3 条以上明显成立：

- 默认降级为 `类型样本 / 参照物`
- 不默认进入 `重点预期差研究对象`

## 六、进入观察名单前的最低标准

以下 6 条里，至少稳稳答上 4 条，再进观察名单：

- [ ] 它是产业链里的真关键环节
- [ ] 高摩擦 / 替代 / 安全逻辑能清楚传导到它
- [ ] 它更像持续能力，不是一锤子题材
- [ ] 客户和验证路径比较清楚
- [ ] 市场预期还没完全打满
- [ ] 最终有机会转成收入、利润和估值改善
