# 投资翻译层

说明：

- 这份文件是：
  `从逻辑基点与工作假设走到可观察市场语言的唯一主文件`
- 它不负责直接给动作。
- 它负责把：
  `世界观 / 主题判断`
  翻成：
  `该看什么 / 先验证什么 / 市场是否已经在交易这件事`

一句话：
`先翻译，再执行。`

## 这层回答什么

当你已经有：

- 逻辑基点
- 工作假设

接下来先回答三件事：

1. 这件事会先影响哪些市场变量
2. 哪些变量只是背景，哪些变量是真正的验证入口
3. 市场当前交易的是事实、预期变化，还是情绪叙事

## 这层不回答什么

它不直接回答：

- 现在要不要下单
- 仓位多大
- 是继续观察还是小仓试探

这些属于：

- `PORTFOLIO_ROLE_BUDGET.md`
- `../playbooks/ACTIONS.md`

## 默认翻译顺序

固定按这 4 步走：

1. `逻辑基点`
2. `工作假设`
3. `投资翻译`
4. `执行规则`

其中，这一层只负责第 3 步。

## 当前常见翻译对象

### 1. 宏观或地缘判断

先翻成：

- 指数
- 商品
- 汇率
- 利率
- 风险偏好

对应高频 reference：

- `SIGNALS.md`
- `TIME_HORIZONS.md`

### 2. 主题到对象的翻译

先翻成：

- 受益产业链
- 关键环节
- 热度过滤
- 观察名单

对应高频 reference：

- `DISCOVERY_PIPELINE.md`

### 2.1. 对象到公司层验证的翻译

当主题已经落到具体公司时，
不要只看财报和季报，
还要继续翻成：

- 公司动态
- 产业链前置信号
- 行业与政策催化
- 市场已经交易了多少

对应高频 reference：

- `EQUITY_SIGNAL_STACK.md`

### 3. 时间尺度翻译

先分清：

- 事件交易
- 阶段交易
- 长周期判断

对应高频 reference：

- `TRADE_TYPE_CHECK.md`
- `STAGE_TRADING.md`
- `STAGE_FAILURE.md`
- `STAGE_VS_LONGTERM.md`

## 当前最重要的边界

- 宏观判断不能直接跳交易动作
- 主题判断不能直接跳股票代码
- 市场反应不能只看单一变量
- 一切翻译都先服务于观察和验证，再服务于执行

## 和其他文件的关系

- 这份文件是 `投资翻译` 的唯一主文件
- `SIGNALS.md` 负责变量观察 reference
- `DISCOVERY_PIPELINE.md` 负责主题到对象的发现 reference
- `EQUITY_SIGNAL_STACK.md` 负责公司层前置信号 reference
- `TIME_HORIZONS.md` 与 `TRADE_TYPE_CHECK.md` 负责时间尺度和类型校正
- `PORTFOLIO_ROLE_BUDGET.md` 负责组合预算与角色分工，不在这里重复
- `../playbooks/ACTIONS.md` 负责执行规则，不在这里重复

## 以后怎么落新结论

如果一条新结论主要是在回答：

- 该看哪些变量
- 怎样把宏观翻成市场语言
- 怎样把主题翻成研究对象
- 怎样区分事实、预期和情绪

优先先更新这份文件。

只有当结论已经变成：

- 继续观察
- 小仓位试探
- 试探后再加一点
- 试探后撤回

这类动作规则时，才去 `../playbooks/ACTIONS.md`。
