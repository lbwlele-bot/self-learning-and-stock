# 估值带与动作带

这层回答一个系统此前已经反复碰到、
但还没有独立长成主规则的问题：

- 估值到底怎样翻成动作

一句话：

`估值不是直接买卖按钮，而是要结合角色、时间尺度和验证状态，翻成不同的动作带。`

## 1. 这层回答什么

这层主要回答：

1. `便宜 / 合理 / 偏贵 / 拥挤` 这些话，怎样落成动作语言
2. 为什么同样是“估值贵”，不同角色的仓位不该做同样动作
3. 什么时候更适合：
   - `继续吸纳`
   - `冻结加仓`
   - `兑现一部分`
   - `降级持有`
4. 为什么“估值不便宜”不自动等于应该卖出

## 2. 这层不回答什么

这层不直接回答：

- 某只股票现在的精确合理价格
- 静态 PE 到多少一定买、多少一定卖
- 不同赛道的严格估值公式
- 没有结合逻辑和角色时的机械动作

这些仍然需要回到：

- `LONGTERM_JUDGMENT.md`
- `TIME_HORIZONS.md`
- `ACTIONS.md`
- 具体对象自己的研究卡与持仓记录

## 3. 为什么现在需要这层

系统此前已经有：

- `估值中枢`
- `分层吸纳`
- `退出规则`

但中间还缺一座桥。

如果没有这层，就很容易出现下面几种常见错误：

- 觉得便宜，就想当然地买
- 觉得贵了，就想当然地卖
- 看到估值高，就忘了市场可能在交易更强的下一段逻辑
- 看到估值低，就忘了它也可能是在交易逻辑变差

所以这层真正要解决的不是：

- `估值高不高`

而是：

- `当前估值状态，应该对应什么动作带`

## 4. 先记住一个核心原则

估值本身不直接产生动作。

更合理的顺序是：

1. `先看角色`
2. `再看逻辑和验证状态`
3. `最后才看估值带`

也就是说：

- `便宜`
  不自动等于
  `继续吸纳`
- `贵`
  不自动等于
  `全部撤回`

动作永远是：

- `角色`
- `逻辑`
- `估值`

三者一起决定。

## 5. 第一版先分五种估值动作带

第一版不追求复杂模型，
先按最实用的五种状态处理。

### A. 便宜但逻辑未稳

更接近：

- 看起来不贵
- 位置已经回落不少
- 但主逻辑、验证和承接还没稳

默认动作：

- `继续观察`
- 或 `小仓位试探`

不默认动作：

- `分层吸纳`

因为这里最大的风险是：

- 你买的不是便宜
- 而是还没跌完的逻辑问题

一句话：

`便宜但未稳，优先观察，不急着吸纳。`

### B. 便宜且逻辑未坏

更接近：

- 估值或位置已回到更可承受区间
- 跌因更像情绪、风格、流动性或短期验证停顿
- 主逻辑并没有明显破坏

默认动作：

- `分层吸纳`
- 或 `继续持有`

这类情况最适合：

- 已有研究基础
- 愿意中期持有
- 且对象本身适合建仓

一句话：

`便宜且逻辑未坏，才有资格进入继续吸纳。`

### C. 合理区

更接近：

- 估值不算特别便宜
- 也还没到明显拥挤
- 市场对它的预期大致处在可接受区间

默认动作：

- `继续持有`
- `按验证决定是否再加一点`

这里最容易犯的错是：

- 因为它“不便宜”，就错过本该继续参与的对象

一句话：

`合理区最常见的正确动作，不是折腾，而是继续让事实说话。`

### D. 偏贵但逻辑继续强化

更接近：

- 估值已经不舒服
- 但新订单、新产品、新预算、新盈利、新市场份额仍在继续强化
- 市场不一定只是情绪过热，也可能是在提前交易下一段

默认动作：

- `继续持有`
- `冻结加仓`

特殊情况下，仍可保留小幅升级空间，
但前提必须是：

- 新事实继续显著强化
- 而不是你单纯更想追

一句话：

`偏贵但逻辑更强，优先冻结加仓，而不是急着全卖。`

### E. 昂贵且乐观打满

更接近：

- 估值已经明显贵
- 情绪拥挤
- 市场对中期乐观已经提前打得很满
- 后续需要很强的事实兑现才能继续支撑

默认动作：

- `兑现一部分`
- `降级持有`

如果这时再叠加：

- 角色漂移
- 仓位涨出来过大
- 对象自身开始弱于应有状态

就更应优先考虑：

- `降级持有`

一句话：

`昂贵且乐观打满，默认不再加，开始考虑兑现或降级。`

## 6. 一个最小动作表

如果当前更像下面这些情况，第一版默认先翻成：

- `便宜但逻辑未稳` -> `继续观察 / 小仓位试探`
- `便宜且逻辑未坏` -> `分层吸纳 / 继续持有`
- `合理区` -> `继续持有 / 按验证决定是否再加一点`
- `偏贵但逻辑继续强化` -> `继续持有 / 冻结加仓`
- `昂贵且乐观打满` -> `兑现一部分 / 降级持有`

补充边界：

- `全部撤回`
  默认不由“估值高”单独触发
- 它更应该来自：
  - `逻辑被证伪`
  - `角色失效`
  - `市场持续证明原判断不成立`

## 7. 不同角色，要用不同估值语言

这一步比“估值高低”本身更重要。

### A. 进攻型成长 / 第二波承接仓

更接近：

- `国电南瑞`
- 未来其他成长或承接型核心对象

这类对象更该防的是：

- `估值拥挤`
- `预期跑太快`
- `增长质量跟不上`

默认动作翻译：

- `便宜且逻辑未坏`
  -> `分层吸纳`
- `合理区`
  -> `继续持有`
- `偏贵但继续强化`
  -> `冻结加仓`
- `昂贵且乐观打满`
  -> `兑现一部分 / 降级持有`

这里最重要的一句是：

`成长和承接型对象，贵不一定该卖，但贵到需要更强事实去接时，动作就该收敛。`

### B. 稳定器 / 高股息 / 类债权益仓

更接近：

- `长江电力`

这类对象更该防的是：

- `类债资产集体重定价`
- `高股息拥挤`
- `仓位涨出来后超出稳定器角色`

默认动作翻译：

- `便宜且逻辑未坏`
  -> `继续持有 / 小幅吸纳`
- `合理区`
  -> `继续持有`
- `偏贵`
  -> `冻结加仓`
- `昂贵且拥挤`
  -> `兑现一部分 / 再平衡 / 降级持有`

这里最重要的不是问：

- `还能不能大涨`

而是问：

- `它还值不值得继续当稳定器`

### C. 对冲 / 安全垫仓

更接近：

- `黄金`

这类对象本身就不适合只用静态估值语言看。

更应该看的不是单纯 PE 或 PB，
而是：

- `拥挤不拥挤`
- `美元和利率是否强压`
- `对冲需求是否还在`
- `市场是不是先交易满了风险叙事`

默认动作翻译：

- `对冲需求仍在 + 只是短期不顺`
  -> `继续持有 / 冻结加仓`
- `风险叙事已经被市场打满`
  -> `兑现一部分或降回战略入口`
- `对冲需求显著下降 + 修复持续偏弱`
  -> `降级持有`

这里最重要的一句是：

`黄金更像看对冲价值和拥挤度，而不是看“贵了没有”。`

### D. 事件转研究观察仓

更接近：

- `中海油服`

这类对象更该防的是：

- 你误把事件溢价当成长周期估值抬升

默认动作翻译：

- `事件溢价退潮，但公司逻辑在补`
  -> `冻结加仓 / 保留观察仓`
- `事件溢价退潮，且公司逻辑接不上`
  -> `全部撤回`

这里最重要的一句是：

`对事件转研究对象，估值不是主轴，逻辑切换能不能成立才是主轴。`

## 8. 和现有规则怎么衔接

这层先回答：

- `当前估值状态更对应哪条动作带`

然后再回到：

- `ACTIONS.md`

也就是说：

- `ACTIONS.md`
  负责动作词汇本身
- 这份文件
  负责把估值语言翻成动作偏向

### 一个简单顺序

以后遇到“便宜了要不要补”“贵了要不要收”时，
优先按这 4 句走：

1. `它是什么角色`
2. `主逻辑有没有坏`
3. `当前更像哪一种估值带`
4. `对应动作应收敛还是应扩张`

## 9. 当前边界

- 这不是赛道估值教科书
- 这不是严格财务模型
- 这不是一步到位的精确定价系统
- 这不是替代对象研究卡

它当前只负责：

- `把估值概念翻成更可执行的动作带`

## 10. 当前一句话版本

`估值真正有用的地方，不是告诉你“它贵不贵”，而是告诉你：在当前角色和逻辑状态下，你的动作应该扩张、冻结，还是开始收。`
