# 深搜检查目录

定位：

- `默认搜索质量标准`

适用场景：

- 任何涉及事实判断、时事、宏观、公司、行业、货币、资产的话题
- 长期观察标的
- 不满足于主流新闻和财报
- 需要尽可能抓到更细的公开前置信号

## 1. 先明确边界

- 只搜索公开可得信息
- 不假装能看到私有、内网、未公开资料
- 默认搜得细，但不默认把搜索树完整展示给用户
- 默认围绕当前主题窄域展开，不预加载未提及主题
- 可以比“明面上的新闻”再往下翻，但必须区分：
  - 官方口径
  - 生态/开发者信号
  - 二级媒体和论坛

## 2. 搜索顺序

### 第一层：官方与硬口径

- 交易所公告 / 年报 / 季报 / 业绩预告
- 公司官网新闻、活动、产品发布
- 投资者交流纪要、互动问答
- 股权激励、经营目标、重大项目

### 第二层：生态与开发者前置信号

- GitHub / Gitee / 开源仓库
- BSP / SDK / 工具链更新
- 开发者论坛、适配记录、教程、demo
- 板卡、模组、方案商发布页

### 第三层：场景与客户信号

- 展会资料、行业案例、合作伙伴页面
- 方案商和客户的公开产品页
- 招聘方向和组织动作
- 行业垂直媒体

### 第四层：辅助验证

- 主流财经媒体、券商研报
- 论坛与社区讨论
- 二次转引

## 3. 默认输出方式

- 先给结论
- 再给 2-4 条关键事实支撑
- 最后给投资含义
- 不默认展开完整搜索过程
- 只有当事实冲突大、用户要更细、或高风险问题需要证据链时，才展开说明来源层次

## 4. 输出时必须说明

- 哪些是已核实事实
- 哪些是基于公开信号的推断
- 哪些只是弱信号，不能下结论

## 5. 使用提醒

- 这份清单描述的是默认搜法，不是额外模式。
- 新对话不需要再额外提醒“做深搜”，除非用户想明确看搜索层次和证据链。
