# 当前系统的下一阶段重点

接下来不再把主要精力放在短期事件交易上。

下一阶段重点是：

1. 阶段交易
2. 长周期判断

## 为什么

因为当前系统已经有了：

- 事件交易的识别
- 预警模式
- 最小动作规则

但更适合长期建设的，还没有真正展开：

- 阶段交易框架
- 长周期判断框架

## 下一阶段最值得补的两个问题

### A. 阶段交易

需要回答：

- 一轮变化怎样才算“阶段性成立”？
- 扩散、持续性、承接力怎么观察？

当前已补：

- `STAGE_TRADING.md`

### B. 长周期判断

需要回答：

- 什么样的变化才算“长期逻辑真的变了”？
- 政策、盈利、估值、经济之间怎么串起来？

## 当前建议

先补阶段交易，再补长周期判断。

原因：

- 阶段交易比长周期更接近你现在已有的系统基础
- 它也更适合作为短期事件和长期配置之间的桥梁
