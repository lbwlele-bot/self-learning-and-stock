# 特定股票研究卡

说明：

- 这份文件现在更适合作为 `skills/company-card-builder/` 的方法 reference
- 高频入口优先走 `skills/README.md`
- 个股层研究不应只依赖财报，前置信号框架见：
  `../frameworks/EQUITY_SIGNAL_STACK.md`

这个目录只放：
`针对单一公司 / 单一股票的研究卡`

它不承担系统总导航功能，也不替代：

- `TOPIC_MAP.md`
- `memory/logic-bases.md`
- `docs/frameworks/DISCOVERY_PIPELINE.md`

更合适的用法是：

1. 先在主题层或发现路径里形成研究方向
2. 再把具体公司压成一张可复用研究卡
3. 后续围绕这张卡持续补充验证、修正和反驳

一句话：
`这里是公司层研究档案，不是系统入口层。`

## 研究卡建议结构

每张卡尽量只保留这些内容：

1. 公司定位
2. 为什么进入系统
3. 业务与利润来源
4. 当前价格 / 估值 / 波动
5. 支持逻辑的证据
6. 关键验证信号
7. 反方意见 / 主要风险
8. 当前结论

## 目录规则

- 每家公司一个文件
- 文件名格式：
  `代码-公司名.md`
- 如果只是临时提及，不建卡
- 只有进入持续跟踪状态的公司，才进入这个目录
