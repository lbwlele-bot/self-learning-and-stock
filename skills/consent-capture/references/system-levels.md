# System Levels

Use these levels before picking a file.

## 1. Discard

Use when the content is:

- only emotional reaction
- wording with no durable value
- duplicate with no improvement
- not yet clear enough to save

Result:

- do not write

## 2. Artifact Layer

Use when the content is local to:

- one conversation
- one market day
- one historical replay
- one company card
- one case note

Typical homes:

- `../../sessions/`
- `../../docs/cases/`
- `../../docs/stocks/`

This layer stores evidence, examples, and local notes.

It should not become a hidden second truth layer.

## 3. Topic Layer

Use when the content changes the current state of one topic.

Typical signs:

- a topic status changes
- a working hypothesis is added or refined
- the current execution expression of a theme changes

Primary home:

- `../../TOPIC_MAP.md`

## 4. System Layer

Use when the content improves the repo's durable backbone.

Typical signs:

- it changes the long-term worldview anchor
- it changes how future reasoning should be done
- it updates the current system stage
- it adds a durable open question

Primary homes:

- `../../memory/logic-bases.md`
- `../../memory/principles.md`
- `../../memory/context.md`
- `../../memory/open-questions.md`

## 5. Method Layer

Use when the content is neither a worldview anchor nor a topic update, but a reusable way of working.

Typical signs:

- a repeatable filter
- a reusable bridge from macro to market
- a reusable research checklist
- a reusable execution rule

Primary homes:

- `../../docs/frameworks/`
- `../../docs/playbooks/`

## Selection Rule

Prefer the lowest layer that fully preserves the value of the idea.

Do not promote upward just because the sentence sounds important.
