# Discovery Pipeline

Use this skill to convert a topic into a researchable beneficiary chain. It is the default bridge between thematic reasoning and concrete watchlist candidates.

## Read Order

- `../../TOPIC_MAP.md`
- `../../memory/logic-bases.md`
- `../../memory/context.md`
- `../../docs/frameworks/DISCOVERY_PIPELINE.md`
- `../../templates/key-link-watchlist-template.md`

Use `references/current-theme-heuristics.md` when the topic is `high-friction world / dollar weaponization / A-share expectation gap / retail niche`.

## Pipeline

Always walk the full chain:

1. `logic base`
2. `working hypothesis`
3. `beneficiary chain`
4. `key link`
5. `heat filter`
6. `watchlist`

Do not jump straight from theme to stock code.

## Heat Filter Rule

This skill must distinguish:

- `good company`
- `good research object`

Default demotions:

- fully crowded leaders
- pure concept names
- names driven mostly by headlines and trading heat
- objects with weak profit transmission

## Current Mainline Heuristic

For the current theme, prefer objects that look like:

- beneficiaries of internal budget reallocation
- safety / substitute / resilience links
- key links whose importance can travel into orders and profit
- companies still partly covered by external risk discount

Do not prioritize the hottest dragon by default.

## Output Contract

Return:

1. `topic -> hypothesis -> chain`
2. `key-link judgment`
3. `heat / expectation saturation check`
4. `watchlist candidates`
5. `why it is not yet a trading object`

If persistence is requested, write the candidate using the watchlist structure before promoting it to a company card.
